Things that all causes the crash to not happen

- Removing CookieManager.getInstance() from onPostExcute()
- Move CookieManager to before starting AsyncTasks
- Disabling encryption on the Realm
- Only happens on real devices. Seen on Nexus 5 and Nexus 7 with Android 5.1 and Android 5.0.2
- Only having one RealmObject with 10 fields.
- Only having 1 field in 15 RealmObjects
- Removing SystemClock.sleep(), but not always it makes the crash come and go.
- Run same code on x86 emulator
- Run same code on arm emulator

What doesn't remove the crash
- Having @PrimaryKey on all objects doesn't make a difference

Current thesis:
- Race condition somewhere.
- CookieManager calls down to Chromimum CookieManager on Android 5+, which also uses OpenSSL. Somehow our usage conflicts
  with theirs.


Missing tests:
- Run project on an older actual device.
